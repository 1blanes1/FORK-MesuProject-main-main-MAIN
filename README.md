Путь к html'кам - json_api\api\templates\app
вы работаете в этой папке
1. все html'ки остаются в этой папке
2. создаются папки css и js и в них уже пишутся соответственные блоки 
3. если не знаешь, что возвращает api смотри api_syntax.md
4. если в api_syntax не написано, то идешь в файл json_api\json_api\urls.py
5. там в path() второй элемент ссылкается на функцию, которая вызывается при обращении на этот api
6. если и так не понял, пиши мне
7. для запуска сервера python manage.py runserver ВАЖНО: вводить команду надо из папки где находиться manage.py т.е json_api 
8. Знакомимся с новыми технологиями DOCKER. Из директории где лежит Dockerfile запускаешь 
```1
docker build -t my-django-app .
```
```2
docker run -d -p 8000:8000 --name django-app my-django-app
```
^ вот так два раза в день делать будешь, локальный питон тебе спасибо скажет

```
http://127.0.0.1:8000/
```
^ здесь свой сайт найдёшь
Если спросишь, почему переписал html но всё осталось как было, то это потому что ты изменил в своей директории а не в контейнере


```
docker-compose up --build
```
